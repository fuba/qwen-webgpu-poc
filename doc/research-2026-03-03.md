# Research Notes (March 3, 2026): Running Qwen3.5-0.8B with WebGPU in Browser

## Conclusion

- `Qwen/Qwen3.5-0.8B` is the base model distribution and does not provide the
  browser-ready ONNX layout by itself.
- `onnx-community/Qwen3.5-0.8B-ONNX` includes required ONNX artifacts and
  tokenizer files for browser inference.
- `transformers.js` has implementation support for `qwen3_5` in newer builds.
- Mapping-wise, `qwen3_5` is handled through image-text-to-text mappings in the
  updated registry path used by this PoC.
- Simon Willison's local-folder approach (`env.customCache`) can be adapted to
  this PoC architecture.

## Implementation outcome in this repository

- The PoC was completed with local-folder loading on top of `env.customCache`.
- Chat UX includes reset/interrupt behavior without model reload.
- `<think>` segments are split from final answer text and rendered separately.
- Markdown rendering is enabled for assistant and user messages.
- Image prompts are supported; large images are resized before inference
  to reduce WebGPU OOM failures.

## Implementation techniques and design choices

- **Worker-first model lifecycle**
  - Model/processor loading is isolated in `worker.js` so UI rendering remains responsive.
  - A warmup generation (`max_new_tokens: 1`) is executed after load to pre-compile
    WebGPU shaders and reduce first-response latency spikes.

- **Local-folder model loading without backend**
  - Selected files are mapped into `env.customCache.match()` with normalized paths.
  - Multiple lookup paths (`onnx/...`, relative paths, basename fallback) are used
    to absorb folder-layout differences in user-selected directories.
  - `ModelSingleton.reset()` is called before loading new local files to avoid stale sessions.

- **Safe multimodal prompt construction**
  - The message-to-model adapter keeps image placeholder order aligned with uploaded image order.
  - Empty text with image is converted to a deterministic default prompt
    (`Describe this image.`) to avoid ambiguous requests.

- **WebGPU stability against OOM**
  - Uploaded images are decoded in the worker and constrained by max edge and max area
    before processor/model execution.
  - For image prompts, generation length is reduced (`max_new_tokens: 256`) to cap runtime
    memory pressure.
  - OOM-like failures are remapped to user-facing guidance instead of raw runtime dumps.

- **Chat UX robustness**
  - `Reset chat` during generation uses an interrupt-then-reset flow to prevent
    interleaving new/old state.
  - IME handling checks composition state and `keyCode=229` to prevent accidental submit
    while confirming Japanese input.
  - Saved folder summary is persisted in `localStorage`, while file handles are not persisted
    (browser security model), then reflected as `Load local model` UX on revisit.

- **Output rendering quality and safety**
  - `<think>...</think>` is parsed into a separate low-emphasis block for readability.
  - Markdown rendering uses `marked` + `DOMPurify` to preserve formatting while sanitizing HTML.

- **Regression control**
  - Unit tests cover input submit behavior (including IME), chat status states,
    local-folder storage, multimodal message conversion, think-tag parsing,
    and image size constraints.
